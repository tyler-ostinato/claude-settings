# Project Instructions

## General Workflow
- Before starting, read existing code in the area you're changing to understand conventions in use
- Run the project's existing test suite before finishing any logic change — check `Makefile`, `package.json` scripts, or README for the test command
- Follow the code style already present in the file — don't impose a different style

## Making Changes
- Keep diffs small and focused; one logical change per commit
- Don't refactor or reformat code outside the scope of the task
- If a build or lint command exists (e.g. `make`, `make lint`), run it before finishing

## When Uncertain
- Check the README or existing docs before asking
- Look at how similar problems are solved elsewhere in the codebase
